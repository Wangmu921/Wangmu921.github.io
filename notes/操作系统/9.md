# 操作系统9：地址转换

## 一、 内存虚拟化的核心驱动力：高效率、控制与灵活性

内存虚拟化与 CPU 虚拟化的设计哲学极其相似，其核心目标是兼顾**高效率（Efficiency）**与**强控制（Control）**，同时赋予程序极大的**灵活性（Flexibility）**。

1. **效率与控制的硬件支撑**：
   * **MMU（内存管理单元，Memory Management Unit）**：这是 CPU 内部的核心硬件。它的主职是负责最快速的**地址转换（Address Translation）**，将程序发出的虚拟地址秒级转换为物理内存的绝对地址。
   * **TLB（转换后备缓冲，Translation Lookaside Buffer）**：硬件缓存，用于加速虚拟地址到物理地址的转换过程，防止每次内存访问都去查询内存中的页表。
   * **异常机制（Exceptions）**：当用户进程尝试执行非法内存操作时，硬件会自动触发异常（如 `page fault` 缺页异常、`protection fault` 保护异常），强行切回内核态。
2. **软件的灵活性表现**：
   * 程序在运行期间，看到的完全是一个**连续、私有、统一且从 `0` 开始**的巨型虚拟地址空间。
   * 在幕后，对应的物理内存**完全不必连续存放**。程序可以自认为拥有一个巨大且干净的独占地址空间，极大简化了编译器的代码生成和链接过程。

## 二、 内存虚拟化的三大基本假设（简化的理想国）

为了深入浅出地推演地址转换的硬件逻辑，本章首先引入了三个极其简单、在现实通用系统中并不成立的**初始假设**：
* **假设 1**：每个用户进程的虚拟地址空间在物理内存中是**绝对连续存放的（Contiguously）**。
* **假设 2**：进程地址空间的总大小**不会太大（Not too big）**，显著小于当前计算机的物理内存总量。
* **假设 3**：系统中所有并发进程的**地址空间大小是完全相同的（The same）**。

## 三、 动态重定位：基址与界限机制 (Base and Bound)

为了在上述假设下落地地址转换，科学家发明了**基址与界限机制（Base and Bound）**，也称为**动态重定位（Dynamic Relocation）**。它依赖 CPU MMU 内部引入的两个核心物理特权寄存器：

[Image of MMU base and bound address translation hardware mechanism with bounds check]

### 1. 硬件寄存器职责定义
* **基址寄存器（Base Register）**：存放当前正在运行进程在物理内存中的**绝对起始物理地址**。
* **界限寄存器（Bound / Limit Register）**：存放该进程虚拟地址空间的**物理大小上限**（或者存放该进程允许访问的最大合法虚拟地址）。

### 2. 地址转换的硬核逻辑与公式
每当进程执行一条尝试访问虚拟地址（Virtual Address, VA）的物理指令（如加载、保存、或者获取指令）时，硬件 MMU 会自动在底层执行以下两步电路盘点：
1. **硬件安全检查（防止踩踏）**：
   $$\text{检查 } VA \ge 0 \quad \text{且} \quad VA < \text{Bound}$$
   * **异常触发**：如果进程发出的虚拟地址越界（例如 $\text{VA} \ge \text{Bound}$），硬件 MMU 会立刻断开内存流水线，**强行抛出系统的“非法内存访问（Illegal Memory Access）”硬件异常中断**，转入内核态，由操作系统接管并将违法程序直接杀死。
2. **物理地址计算转换**：如果检查通过，虚拟地址将通过加法器，平滑转换为流向物理内存总线的物理地址（Physical Address, PA）：
   $$\text{PA} = \text{VA} + \text{Base}$$
